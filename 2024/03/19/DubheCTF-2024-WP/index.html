Wecat

分数：370

一个网页版的即时通讯程序。源码：https://github.com/imzusheng/Wechat-Wecat
package.json里指定了服务的启动方式，Dockerfile里使用dev方式启动。

这种启动方式node可以热加载，就是说在服务运行过程中修改js文件的话，node服务会自动重启来加载更新。
对于node来说，webshell不能像php那样上传一个js文件就可以。需要覆盖原来的js文件并在启用了热加载的情况下才能加载恶意代码。
upload.js关键代码：没有过滤，可以目录穿越，存在任意文件上传（覆盖）。

综上，我们可以上传恶意js文件覆盖源文件来重写相关web路由，相当于代码执行。
可以尝试覆盖server\server\src\route\commonRouter.js

Master of Profile

分数：512

一个C++写的程序。
官方文档：https://github.com/tindy2013/subconverter/blob/master/README-cn.md
路由(main.cpp)：

读文档可知/getprofile路由可以执行任意javascript代码。

enable_filter
设置为所有节点使用自定义的js代码进行筛选
当值为 true 时, 为所有节点使用自定义的js代码进行筛选, 为 false 时不使用。
filter_script
为所有节点使用自定义的js函数进行筛选
可设置为js代码内容，也可为本地js文件的路径
js函数包括一个参数，即一个节点，函数返回为true时保留该节点，返回为false时丢弃该节点
例如:
仅保留加密方式为chacha20的节点
filter_script=function filter(node) {\n if(node.EncryptMethod.includes(‘chacha20’))\n return true;\n return false;\n}

简单来讲就是enable_filter传参为true，filter_script传参为js代码，写一个filter(node)函数，服务器会将节点信息用参数node传入filter函数，判断此节点是否过滤。

服务器使用quickjs作为javascript引擎。没有child_process那些nodejs里的那些module，但是quickjs内置了std和os模块。官方文档：https://bellard.org/quickjs/quickjs.html#os-module

The os module provides Operating System specific functions:
The OS functions usually return 0 if OK or an OS specific negative error code.
Available exports:
exec(args[, options])Execute a process with the arguments args. options is an object containing optional parameters:

在os模块中存在exec函数。可以rce。不过文档似乎很久没更新了，有些语法不对，通过不断本地试错得出exp：

有个坑，用bash没反应，可以读/etc/shells看看，系统有一个ash，通过这个shell可以执行命令。
ash好像没办法反弹shell，也有可能是不出网，所以可以把flag写到/tmp/res里，再通过getlocal读取/tmp/res。

访问/getprofile路由时需要传参token。
在/getlocal路由可以读取任意文件，所以可以通过getlocal读取配置文件拿token。
payload:/getlocal?name=/etc/passwd

在官方文档中关于配置文件的部分：

配置文件
关于 subconverter 主程序目录中 pref.ini 文件的解释，其余格式的配置文件不再赘述，与之相仿。

注：本部分内容以本程序中的 [pref.example.ini](https://github.com/tindy2013/subconverter/blob/master/base/pref.example.ini) 或 [pref.example.yml](https://github.com/tindy2013/subconverter/blob/master/base/pref.example.yml) 或 [pref.example.toml](https://github.com/tindy2013/subconverter/blob/master/base/pref.example.toml) 为准，本文档可能由于更新不及时，内容不适用于新版本。
加载配置文件时会按照pref.toml、pref.yml、pref.ini的优先级顺序加载优先级高的配置文件

经测试题目服务器使用的是pref.yml。
除此之外，/getprofile还需要传入节点文件的文件名，毕竟就是要对这个文件的内容进行排序嘛，项目存在一个例子文件profiles/example_profile.ini，经测试题目环境中也存在这个文件。

bot.py

bot会读取flag然后发往http://localhost:8080/。但奇怪的是服务器根本没有相关处理逻辑，而http code 418也是一个不存在的状态码。

admin.py

访问/admin时会读取环境变量里的哈希后的password。但奇怪的是无论是/proc/self/envrion和.env文件都读不到哈希后的password。
可以读/proc/self/maps。
在/static读任意文件的时候，可以通过range header头指定读取多少字节，感觉可以通过/proc/self/mem读内存去拿各种key什么的。
但mem fd等读取的时候报500错误，可能是没权限读取，到这卡住了，怀疑读.git时没读完整。
